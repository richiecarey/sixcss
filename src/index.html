What is sixCSS?

sixCSS is a simple six column CSS flex-box based responsive grid system.

Installation

First, download and unzip the sixCSS.zip file.

You will see two directories (/css and /sass) and one file (grid_demo.html)

Upload the /css folder to your website and add the following reference to the <head> block of your HTML document. 

<link href="css/main.css" rel='stylesheet' type="text/css">

You're now ready to add any of the following sixCSS classes to your HTML page elements.

Wrapper

Sass developers: the .wrapper class is defined in the /sass/layout/_wrapper.scss file.

The .wrapper class is typically added to a <div> element that encloses all of the content of the page. 

<body>
   <div class="wrapper">
    ...EVERYTHING ELSE ON THE PAGE...
   </div>
</body>

Grid

Sass developers: the .grid class is defined in the /sass/layout/_grid.scss file.

To create a grid, use a <div> element with a class of .row and nested <div> elements with class attribute values of col-1, col-2, col-3, col-4, col-5, or col-6 as columns within that row.

For example, to display a six column grid you'd nest six <div> elements within a row, each with a class of .col-1 like this... 

<div class="row">
  <div class="col-1"><h4>.col-1</h4></div>
  <div class="col-1"><h4>.col-1</h4></div>
  <div class="col-1"><h4>.col-1</h4></div>
  <div class="col-1"><h4>.col-1</h4></div>
  <div class="col-1"><h4>.col-1</h4></div>
  <div class="col-1"><h4>.col-1</h4></div>
</div>

Header

Sass developers: the .header class is defined in the /sass/layout/_header.scss file.

The .header class is applied to the <header> tag to provide basic styling to the header. 

<header class="header">
   ...
</header>

Nav

Sass developers: the .nav_primary class is defined in the /sass/layout/_nav_primary.scss file.

The .nav_primary class is applied to a <nav> element to provide basic styling for the main, often top, horizontal menu. 

<nav class="nav_primary">
<ul>
  <li><a href="#menu1">Page 1</a></li>
  <li><a href="#menu2">Page 2</a></li>
  <li><a href="#menu3">Page 3</a></li>
  <li><a href="#menu4">Page 4</a></li>
</ul>
</nav>

The .nav_secondary class is applied to a <nav> element to provide basic styling for a secondary vertical menu. 

<nav class="nav_secondary">
<ul>
  <li><a href="#menu1">Page 1</a></li>
  <li><a href="#menu2">Page 2</a></li>
  <li><a href="#menu3">Page 3</a></li>
  <li><a href="#menu4">Page 4</a></li>
</ul>
</nav>

Sass developers: the .nav_secondary class is defined in the /sass/layout/_nav_secondary.scss file.

Jumbotron

Sass developers: the .jumbotron class is defined in the /sass/components/_jumbotron.scss file.

To transform your standard header to a Jumbotron, wrap you header items in a <div> and assign a class attribute value of .jumbotron 

<header class="header">
   <nav class="nav_primary">
     <ul>
       <li><a href="#menu1">Page 1</a></li>
       <li><a href="#menu2">Page 2</a></li>
       <li><a href="#menu3">Page 3</a></li>
       <li><a href="#menu4">Page 4</a></li>
     </ul>
   </nav>
   <div class="jumbotron">
      <div class="row">
         <div class="col-4">
            <h1>sixCSS</h1>
            <p>A Six Column CSS Grid System</p>
         </div>
         <div class="col-2">
            <div class="align-middle">
               <a href="#" class="button button-secondary button-large button-x-large">Download</a>
            </div>
         </div>
      </div>
   </div>
</header>

Card

Sass developers: the .card class is defined in the /sass/components/_card.scss file.

The .card class can be added as an style attribute to provide some basic styling to block level elements. Typically added to a <div> element, the .card class can be useful for adding a bit of definition to grid elements. 

<div class="row">
   <div class="col-6-medium card"><h4>col-6</h4></div>
</div>

Buttons

Sass developers: the various button classes are defined in the /sass/components/_buttons.scss file.

The .button class can be added to a <button> or <a> HTML tag. 

<button class="button">.button class added to a button tag</button>
<a href="#" class="button">.button class added to an a tag</a>
.button class added to an a tag

Visual enhancement classes can also be added alongside the .button class to change button appearance: 

<button class="button button-primary">button button-primary</button>
<button class="button button-secondary">button button-secondary</button>

There are also options to increase the size of the buttons: 

<button class="button button-primary button-large">button button-primary button-large</button>
<button class="button button-primary button-x-large">button button-primary button-x-large</button>

Code

Sass developers: the .code class is defined in the /sass/components/_code.scss file.

The .code class is applied to the <code> tag to provide styling for the display of code snippets. 

<code class="code">
   <p>This is a snippet of HTML code.</p>
</code>

Footer

Sass developers: the .footer class is defined in the /sass/layout/_footer.scss file.

The .footer class is applied to the <footer> tag to provide basic styling to the footer. 

<footer class="footer">
   <p>Footer Text Goes here</p>
</footer>

Breakpoints

Several breakpoints are defined in /sass/utils/_variables.scss: 

$small: 450px;
$medium: 600px;
$large: 900px;
$x-large: 1200px;

Colors

Primary and secondary colors along with several standard colors are defined in /sass/utils/_variables.scss: 

$primary-color: #8c2840;
$secondary-color: #32826f;
$white: #fff;
$black: #000;
$red: #f00;

Typography

Typography import is defined in /sass/base/_typography.scss 

@import url('https://fonts.googleapis.com/css?family=Crimson+Text|Open+Sans|Source+Code+Pro');

Fonts are defined in /sass/utils/_variables.scss: 

$serif: 'Crimson Text', serif;
$sans-serif: 'Open Sans', sans-serif;
$mono: 'Source Code Pro', monospace;
